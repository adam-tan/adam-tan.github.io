    快速上手

    这一部分简单介绍了支付后台(bm)项目的代码结构、基本写法，目的是让有一定react基础，但之前没有用过rx+redux+observable全家桶的人，在阅读后也能快速上手，写出符合要求的代码。

    代码结构

    每一个页面都写在containers中，主要包括以下部分：
    建议在一开始想好index.js要哪些东西，搭出架子并逐步完成，其余可按照如下顺序去写：

    • constants.js
      定义了包括actions的type在内的一系列常量

    • actions.js
      export一系列小函数，每个函数都返回一个对象，格式为：

      {
type: CONSTANTS,  //  引入constants中的常量作为type
payload
}

    • reducer.js

    定义了这个页面的state，以及接受到action时对state的同步操作。
    所有的state都是immutable的，注意updateIn, update, set, concat等的用法。

    • epics.js

    定义了这个页面的所有异步流，以流的方式接收异步action，并吐出同步的action。
    用到了rxjs, 注意操作符的选择(switchMap, flatMap)和生成流(forkJoin, of)的操作符。

    • selectors.js

    定义了所有的选择器，利用封装的createSelector方法，从$state中获取需要的selector。

    • styled.js

    定义了DomWrapper为主的样式。利用了’styled-components’。

    • index.js

    页面主体部分，有自己的state。接收所有的actions, selectors作为props。
    所有其余的文件均在这里拼接：

    export default pipe(
  connect(
    createStructuredSelector({
      $infos: $infos_,
      $records: $records_,
    }),
    actions
  ),
  authViewWrapper
)(HrCreateFailedPage)

    数据流向

    示意图如下：

    
+-----------------+    +-----------------------------------+
|                 |    |                                   |
|    component    |    |              component            |
|                 |    |                                   |
+--------^--------+    +-----------------^-----------------+
         |                               |
         |                               |
+----------------------------------------------------------+
|                                                          |
|                          container                       |
|                                                          |
+--------^-------------------------------+-----------------+
         |                               |
         |                               |
+--------+--------+    +-----------------v-----------------+
|                 |    |                                   |
|     selector_   |    |               action              |
|                 |    |                                   |
+---+---------^---+    +-------+-------------------+-------+
    |         |                |                   |
    |         |                |                   |
+---v---------+---+    +-------v------+    +-------v-------+
|                 |    |              |    |               |
|     $state      <----+    reducer   <----+     epics$    |
|                 |    |              |    |               |
+-----------------+    +--------------+    +---------------+

    • selector_： getter/computer var_

      • val_ = createSelector( state_, (state) => state.val ) => val

    • store

      • data => immutable 数据 + reducer setter (同步)

    • epics$: redux 和 redux-observe 结合 rxjs 异步 action var$

      • epic(action$, state$).pipe(…).subscribe(store.dispatch)

    • container

      • props => connect(state_ + action())/HOC

    • component

      • data => props + component/el

    如何组装epics和reducer

    每个单独的epics都在root-epics中被引入，并combine成一个总体的epcis。所以不同页面的流，是同时存在，并起作用的。
    每个reducer也是通过combineReducer方法(redux-immutable)组装成一个总的reducer的。但reducer是异步的。state也是异步的。

    // in  store.js
const middlewares = [
    epicMiddleware,
    routerMiddleware(history),
  ]
  const enhancers = [
    applyMiddleware(...middlewares),
  ]
  const composeEnhancers = window.__REDUX_DEVTOOLS_EXTENSION_COMPOSE__ || compose
  const store = createStore(
    createReducer(),
    fromJS(initialState),
    composeEnhancers(...enhancers),
  )

// in reducer.js
export default function createReducer(asyncReducers) {
  return combineReducers({
    route: routeReducer,
    ...asyncReducers,
  })
}

    实践Q&A

    1. index.js中的组件需要用原始js，但container拿到的Props是Immutable的，怎么办？
    • immutableWrapper
    • selector 更好，在selector中处理数据，可以避免在Index中逻辑过多。
    1. 多个container共享props/actions

    抽出共同的reducer/action/selector/constants 文件，并与自己的状态merge

    1. container的render()如何避免重复渲染

    dom上绑定的事件尽量为func，而不是func()
    注意，尽量写成() => {}的形成，否则可能会每次render都执行

    1. rxjs不能写异步方法， 异步控制在哪里做？（多个异步控制，loading态度）

    rxjs通过吐出多个action来操作。
    哪怕index.js里的方法是async的，await actions 也没有作用。因为actions只是发出事件，Pub-sub模式，处理流不是在这一步。

    1. 建议写一个功能的顺序：constants（定义常量） -> action（定义操作） -> reduce（数据操作修改）和selector（数据获取） -> epics（定义异步操作，可同时发出相关操作 ）-> index(调用action，写selector接收数据）

    2. 建议index文件中，功能函数在上，render渲染在最后，方便查看修改

    Issues

    1. request$ && catch-error$

    如何兼容服务器的各种报错，要不要在requset$中处理错误？

    1. setState到底有多快？

    比如说，先setState({index: 1})， 再去做getList()

    getList = () {
  const {index} = this.state
  getList(index)  //异步方法
}

    参考：https://segmentfault.com/a/1190000016805467#articleHeader6

    1. throttle的问题

    上传回单的时候，如果迟迟不finalize（请求时间过长），则无法上传第二个回单？？即使指定了每次throttle的request$都是不同的值。

    踩坑

    1. antd table word wrap

    控制css换行的是word-break属性, 取值为break-all, keep-all, break-word等。
    但加上后，antd的table中仍然遇到了不换行的问题。后来发现，是因为文字容器为button，按钮不会换行，换成span后就正常换行了。

    1. 页面切换时，willUnMount和location_change事件的顺序

    先发生location_change, 再发生上一个页面的willunmount
    注意不要在这两个事件钩子里撞action。

    1. 检查拼写，reducer和epic里的拼写确保一致。

    4.如何自己写上传文件？
    一个container内部共享一个隐藏态的input，使用onInput事件而不是onChange事件，注意每次传完值后将value置为空字符串。
